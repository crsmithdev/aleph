# Verification Before Completion

## Overview

Claiming work is complete without verification is dishonesty, not efficiency.

**Core principle:** Evidence before claims, always.

**Violating the letter of this rule is violating the spirit of this rule.**

## The Iron Law

```
NO COMPLETION CLAIMS WITHOUT FRESH VERIFICATION EVIDENCE
```

If you haven't run the verification in this message, you cannot say the
work is done. "It looks right" is not evidence. "The diff is minimal" is
not evidence. "The types are correct" is not evidence.

## What counts as evidence

Evidence is observed output from the real system on this machine, right
now, showing the claimed behavior.

Now: verify those assertions about the last commit.
