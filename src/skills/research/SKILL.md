# Research

## When to Use

- Researching topics, comparing options, or investigating unknowns
- Tasks requiring synthesis from multiple sources

## When NOT to Use

- The answer is in the codebase — read the code instead
- A single search or doc lookup will suffice — just do it directly

## Process

1. **Define scope** — write questions as concrete criteria before searching
2. **Parallel search** — fire multiple searches simultaneously
3. **Source evaluation** — prefer primary sources; flag aggregators for verification
4. **Synthesis** — summarize against original questions; note gaps
5. **Capture** — findings worth keeping go in a file, not just in conversation

## Output

Structured markdown: one section per question, source links inline, gaps section at end.

## Done when

- All original questions answered or explicitly marked as gaps
- Sources cited for every claim
- Conflicting evidence noted, not hidden
- Findings captured in a durable location if needed beyond this conversation

## Principles

- Research and report are separate phases — don't blend them
- Fetch full source pages, not snippets
- Never omit conflicting evidence
