00 · Theming

Themes are a one-attribute swap

All color tokens live inside [data-theme-id="…"] blocks. 33 themes (22 dark, 12 light) are bundled — pick one with the picker bottom-right. UI code never references raw hex values — only semantic tokens (--bg-secondary, --accent, etc). To add a new preset, copy the dark block and rewrite its values; nothing else needs to change. 02 · Surfaces

Three-tier depth

Every screen in Aleph is built from three background values. The primary is the floor; secondary is where work happens; tertiary is for hover, inputs, and nested chrome. No drop shadows — depth comes from value steps.

04 · Text

Four greys do all the work

Hierarchy comes from value, not size. Primary is for body and headings. Secondary handles axis labels and supporting copy. Muted lives in metadata and timestamps. Disabled is for what's there but not for now.

--text-primary · #e6e8ec
Body, headings, the thing you're meant to read.
--text-secondary · #b4b8c0
Axis labels, supporting copy, second-tier UI.
--text-muted · #797f8a
Timestamps, captions, helper text, hint copy.
--text-disabled · #4a4f58
Disabled controls and placeholder values.

07 · Spacing & radius
