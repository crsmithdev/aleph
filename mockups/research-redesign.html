Speculative decoding uses a smaller "draft" model to generate candidate token sequences, which are then verified in parallel by the larger target model. This approach exploits the fact that verification is cheaper than generation for autoregressive models. KV Cache Optimization

PagedAttention (vLLM) applies virtual memory paging concepts to KV cache management, reducing memory waste from 60-80% to near-zero. This enables 2-4x higher throughput by fitting more concurrent sequences in GPU memory. The technique stores KV cache in non-contiguous blocks mapped via a page table.

Grouped-Query Attention (GQA) shares key-value heads across query heads, reducing KV cache size by 4-8x with minimal quality loss. Combined with prefix caching for shared prompt prefixes, these techniques are now standard in production serving stacks.

LLM Inference Optimization Techniques

What are the latest approaches to optimizing LLM inference latency?

87 findings 24 threads 156 sources $1.24 cost active
Summary

Research across 24 threads identified five major categories of LLM inference optimization. Speculative decoding achieves 2-3x speedup without quality loss. KV cache compression (PagedAttention, GQA) enables 4x larger batch sizes. Most techniques are complementary and stack multiplicatively, suggesting the optimal serving stack combines all five approaches.

Speculative Decoding

seed

Speculative decoding uses a smaller "draft" model to generate candidate token sequences, which are then verified in parallel by the larger target model. This exploits the asymmetry between generation cost (autoregressive, sequential) and verification cost (parallel forward pass). The key insight is that verification of k tokens costs roughly the same as generating a single token.

DeepMind's original approach achieves 2-3x wall-clock speedup with provably identical output distribution — rejected tokens are resampled from the adjusted target distribution, ensuring no quality degradation. The acceptance rate depends on the alignment between draft and target models.

lightbulb Key Finding
conf
92%

The optimal draft model size is 10-20% of the target model parameters. Larger drafts have higher acceptance rates but slower generation; smaller drafts generate faster but waste verification on rejected tokens.

Recent variants push beyond the original framework: Medusa adds multiple prediction heads to the target model itself, eliminating the need for a separate draft model. Eagle uses feature-level draft prediction from intermediate layers. Staged speculative decoding enables multi-step lookahead with nested draft models.

Recommendation

These options aren't mutually exclusive. A strong hybrid approach: Option B's query list and document view (rich cards, reading-optimized typography, annotated findings) + Option C's thread navigator sidebar for the detail page + Option A's workers page as a dedicated top-level route + Option B's branching timeline as the relationship view (replacing the current DAG graph). Per-question config as a slide-over drawer (Option B) works across all layouts.
