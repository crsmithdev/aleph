# Sketches

Less-than-a-plan-but-more-than-a-TODO. Use when an idea needs enough shape
to be eyeballed (interface, file targets, key heuristics) but doesn't yet
warrant a full plan with steps, owners, and milestones.

Typical contents:
- Problem framing (1–3 paragraphs)
- The proposed change, scoped to specific files
- Heuristics / regex / shapes that need user-sanity-check before code
- Open questions deferred to implementation time

Promote to `docs/plans/` once committed to building.
