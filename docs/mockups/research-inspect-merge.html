Combine Events, Telemetry & Reviews into one tab

Three observability views currently live in three sub-tabs. They overlap heavily — a post-mortem already references metrics; the event log already lists steps and errors that show up in telemetry. These three mockups propose three different ways to merge them into a single tab without dropping any of the existing surface area.

Variant A

Activity dashboard — one canonical scroll

Top KPI strip, latest verdict surfaced inline, then telemetry panels and the event log laid out as a two-column dashboard. Everything visible without sub-tabs; the event log gets a sticky right column so it stays in view while you scroll the panels on the left.

Variant C

Live event log lives in a permanent left rail (compressed); the right side is a canvas with a segmented switcher between Overview, Reviews, Telemetry and Decisions. Reading any panel keeps the live stream in peripheral vision — useful while a session is still running.
