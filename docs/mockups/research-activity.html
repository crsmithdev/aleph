⚠ Implementation gate · event-view fidelity

The merged Activity tab must not lose anything from the current EventsView. Past refactors of this view have dropped event types and detail fields; this gate exists so that does not happen again. Stream 2C cannot land until each item below is explicitly verified.

  • Every event type renders. Today: finding, thread, step, search, fetch, error. The merged event-log column must render all six (filter pills shown above include every one).
  • Every detail field carries forward. formatEventDetail, the expandable-row inspection, and the thread-diff chips for status / priority / backoff / retry transitions all encode specific information — none of it can be silently dropped.
  • Both event sources merge. The current view fuses live SSE with DB-backed steps and findings (so rows that aged out of the 1000-event SSE cache still appear). Keep that merge.
  • Filter parity. filterType (all · finding · thread · step · search · fetch · error) + per-thread filter + free-text search + expanded-row inspection all carry over.
  • Verification. Smoke-load a fixture session with ≥ 50 mixed events and assert each event-type label appears at least once in the rendered DOM. Diff the rendered list against a captured-from-main fixture before merging.
