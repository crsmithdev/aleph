Chart controls — five compact approaches

Replacements for the current page-wide ObsControlBar. Each approach binds to a single chart or chart group, stays compact at idle, expands in place without blocking sibling charts, and keeps the per-chart type toggle (line/bar, donut/bar) on the chart itself. All five are shown around the Daily Invocations by Skill 2-chart panel, with a sibling chart underneath to demonstrate non-interference.

A · Header summary chip → anchored popover

One pill in the chart header. Click to expand.

A single chip in the panel header shows the chart's current scope at a glance — 30d · day · 12 series · 2 filters. Click anywhere on the chip to open a portal-positioned popover containing every control. Idle is one row; expanded is anchored to the chip and does not extend over neighbor charts. Modeled on Grafana's per-panel controls + Linear's compact summary chips.

B · Vertical floating toolbar

Pill column on the chart's edge. Each icon owns one control.

A floating pill column docks to the right edge of the chart. Each icon opens its own narrow popover to the left — time, granularity, series, filters, more. Zero header space, every control discoverable at idle, every popover small. Inspired by Material 3's VerticalFloatingToolbar + Apple HIG popovers.

C · Cascading default + override badge

One control at the page level. Charts only show controls when they diverge.

Keeps a single global control at the top of the page (familiar). Each chart inherits silently — no per-chart chrome at idle. When a chart overrides one or more fields, a small overridden badge appears on its header showing what diverges. Click the badge to edit just that chart, or to reset to page default. Modeled on Grafana per-panel time override + Power BI bookmarks.

D · Filter chips + “+” combobox

Linear / Notion-style chip row. Everything is a chip; nothing is hidden.

Range, granularity, series mode, and filters are all chips above the chart. Each chip opens its own tiny popover when clicked. A trailing dashed “+” chip opens a typeahead combobox to add any control or filter property. State is fully visible at idle, scales horizontally rather than into nested menus.

E · Command-bar typeahead

One ⌘K button. Everything searchable.

The chart header collapses to a single icon button (or keyboard shortcut ⌘K). Activation opens a Linear-style command palette scoped to that chart. Every control — range, granularity, series, filters, even chart type — is a searchable row. Arrow keys navigate, Enter applies, Tab drills into multi-step pickers. Maximum density, keyboard-first.

Synthesis

Which to pick

The five approaches cover the design space. The strongest candidate for Construct is a hybrid of C + A: keep a slim page-level default at the top (so pages with one shared scope stay simple), and put approach A's summary chip on each chart for per-chart overrides — with the override badge from C visible whenever a chart diverges. Pair it with approach E (⌘K) as an opt-in fast path so keyboard users get the typeahead too. Approaches B and D are stronger when filter-heavy exploration is the dominant workflow.
