Combine Knowledge & Process into one tab

Knowledge (concept graph + outline) and Process (threads list + thread map) are different views of the same underlying graph: threads spawn findings, findings compose concepts, concepts link back to threads. These three mockups merge them so you can navigate the run from either side.

Variant D

View-mode switcher — one canvas, five lenses

One large canvas. A segmented control swaps between thread-centric (live, map) and concept-centric (graph, outline) lenses, plus an "Overlay" mode that draws threads and concepts in the same graph with colored edges between them. Filters in a left rail; selected entity inspector slides up from the bottom.

Variant E

Three-pane workspace — tree · canvas · inspector

Left rail is a unified hierarchy (threads with their spawned concepts nested as children). Center is the live canvas — graph, outline, or live findings stream depending on selection. Right is a sticky inspector that adapts to whatever is selected (thread → its findings; concept → its findings + source threads).

Variant F

Run · Mind · Both — least invasive

A top-level segmented control on the new combined tab: Run renders today's Process view (live threads + findings); Mind renders today's Knowledge view (concept graph + outline); Both shows them side-by-side with a synced selection (clicking a thread highlights its concepts and vice-versa). Effectively zero loss of existing functionality.
